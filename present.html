SEO Loop 設計テンプレート

SEOを「1回書いて終わり」ではなく、自律的に回り続けるループとして設計するための穴埋めテンプレート。

使い方: そのままコピーして、対象サイト・指標・停止条件を埋める。エージェントの初期設定やAGENTS.mdの隣に置く。

# SEO Loop 設計テンプレート

## 1. ループの目的（Goal）
- 対象サイト / セクション:
- 改善したい指標:（例: 対象KW群の平均掲載順位 / 対象ページのCTR）
- 目標値と期限:

## 2. トリガー（いつ回すか）
- [ ] cron（例: 毎日 06:00）
- [ ] Search Console のアラート（順位 / CTR 下落）
- [ ] 新規KW発見（他ループからの通知）
- [ ] 手動

## 3. ループ本体（Act -> Observe）
1. Research: 対象KWのSERP・競合上位を分析し、勝ち要件を出す
2. Writer: 要件に沿って新規作成 or 該当箇所を改稿
3. SEO Checker: SEO/GEOスコア・内部リンク・構造化データを検査
4. Reviewer: 事実・E-E-A-T・読みやすさを点検
5. Publisher: CMSへ反映（下書き / 公開）

## 4. 停止条件（Stop condition）
- [ ] 品質スコア >= (閾値)
- [ ] Reviewer の必須指摘がすべて解消
- [ ] 目標順位に到達 or 上限イテレーション（例: 5回）に到達

## 5. 記録（Memory / Log）
- 実行日時・対象・変更点・結果指標を LOG に追記
- 次回はこの LOG を読んでから開始する

## 6. ガードレール
- 公開はレビュー後（人間の最終確認を挟む工程を明記）
- 1回あたりのコスト上限:
⬇ ダウンロード（.md）

💡 停止条件を必ず数値で決めておくと、無限ループ・コスト暴走を防げる。

キーワードリサーチ プロンプト集

検索意図の分類から記事ブリーフ生成まで、Research Agent にそのまま渡せる4本のプロンプト。

使い方: {{ }} の箇所を自分のキーワードに置き換えて、AIエージェントに投げる。

# キーワードリサーチ プロンプト集

## 1. 検索意図の分類
以下のキーワード群を、検索意図（Know / Do / Go / Buy）で分類し、
各意図で狙うべきコンテンツ形式を表にして。
キーワード: {{keywords}}

## 2. 競合ギャップ抽出
対象KW「{{keyword}}」で上位10件の見出し構成を要約し、
多くのページが共通でカバーしているサブトピックと、
逆に手薄なサブトピック（ギャップ）を分けて列挙して。

## 3. ロングテール展開
シードKW「{{keyword}}」から、検索意図が明確でコンバージョンに近い
ロングテールKWを20個、意図の注釈つきで出して。

## 4. コンテンツブリーフ生成
KW「{{keyword}}」の記事ブリーフを作成:
想定読者 / 検索意図 / 推奨文字数 / H2・H3構成案 /
含めるべきエンティティ / 内部リンク候補 / FAQ 5件。
⬇ ダウンロード（.md）

💡 出力はそのまま公開せず、必ず一次データ（実際のSERP）で裏取りしてから使う。

4

記事レビュー チェックリスト

Reviewer / SEO Checker が公開前に毎回通す、E-E-A-T × SEO のチェックリスト。

使い方: 記事ごとにコピーして、各項目をチェックしてから公開する。

# 記事レビュー チェックリスト（SEO × E-E-A-T）

## E-E-A-T
- [ ] Experience: 一次体験・実データ・具体例が入っているか
- [ ] Expertise: 記述が正確で、専門用語の使い方が適切か
- [ ] Authoritativeness: 主張に信頼できる裏付け / 出典があるか
- [ ] Trust: 誇張・断定保証がなく、透明性があるか（最重要）

## 検索意図
- [ ] タイトル・導入が検索意図に一致しているか
- [ ] 読者の「次の疑問」に本文で答えているか

## SEO 技術
- [ ] タイトル / メタディスクリプションが適切（長さ・KW）
- [ ] 見出し階層（H1 -> H2 -> H3）が論理的
- [ ] 内部リンク（関連記事・回遊）が張られている
- [ ] 構造化データ（schema）が適切
- [ ] 画像 alt / 代替テキスト

## 可読性
- [ ] 一文が長すぎない / 箇条書きと散文のバランス
- [ ] 結論が先、根拠が後

## 公開前
- [ ] 事実確認済み（未確認は注記 or 削除）
- [ ] 人間の最終レビュー完了
⬇ ダウンロード（.md）

💡 Trust（信頼性）が最重要。誇張表現を1つでも見つけたら中立表現に直す。

5

AGENTS.md（SEOルール）

SEOループを回すエージェントに読ませる AGENTS.md のSEO運用ルール雛形。

使い方: リポジトリ直下の AGENTS.md にコピーして、自分のサイト事情に合わせて調整する。

# AGENTS.md — SEO運用ルール

## 役割
このリポジトリで動くエージェントは、SEOループ
（収集 -> 分析 -> 執筆 -> 評価 -> 改稿 -> 監視）の一部を担う。

## 執筆ルール
- 検索意図を最初に定義してから書く
- 誇張・断定保証をしない（「必ず上位表示」等はNG）
- 事実には裏付けを付ける。裏が取れない主張は書かない or 注記する
- E-E-A-T（経験・専門性・権威性・信頼性）を満たす

## 品質ゲート（停止条件）
- Reviewer の必須指摘がすべて解消するまで公開しない
- 公開は人間の最終確認後

## メモリ / ログ
- 作業のたびに LOG に「対象・変更・結果指標」を追記する
- セッション開始時は直近の LOG を読んでから着手する

## やってはいけないこと
- キーワードの不自然な詰め込み
- 他サイトの文章の複製
- 出典のない数値・価格・日付の断定
⬇ ダウンロード（.md）

💡 「品質ゲートを通るまで公開しない」を明記すると、自動化しても質が崩れない。

Search Console 改善ワークフロー

Search Console のデータを起点に、順位下落・低CTR・indexエラーを直す監視ループの手順書。

使い方: Monitor エージェントの定期タスクとして登録する。データはAPI / 連携ツール経由で読ませる。

# Search Console 改善ワークフロー

## 前提
Search Console のデータ（順位・CTR・表示回数・index）を
エージェントが読める状態にする（API / 連携ツール経由）。

## 定期チェック（例: 週1）
1. 掲載順位が下落したクエリ / ページを抽出（前週比）
2. 表示回数は多いがCTRが低いページを抽出（改善余地大）
3. index エラー・カバレッジ問題を抽出

## 改善アクション
- 低CTRページ -> タイトル / メタディスクリプションを検索意図に合わせて書き換え
- 順位下落ページ -> 競合の更新点を調べ、該当箇所を改稿（既存記事改善ループへ）
- index エラー -> 原因を分類し、対応 or エスカレーション

## 記録
- 変更前後の指標を LOG に残し、2〜4週間後に効果を再測定する

## 注意
- 一度に多くを変えず、変更点を絞って因果を測れるようにする
- 連携ツールはサードパーティの場合がある。権限は最小限に
⬇ ダウンロード（.md）

💡 変更は一度に絞る。まとめて変えると、何が効いたか分からなくなる。

受け取り方・使い方

  1. 各ブロックの「📋 コピー」でコピー、または「⬇ ダウンロード」で .md を保存（GitHub の kit/ フォルダからも受け取れます）
  2. まず「SEO Loop 設計テンプレート」で目的・停止条件を決める
  3. 「AGENTS.md（SEOルール）」をリポジトリに置き、プロンプト集・チェックリストを各エージェントに渡す
  4. 「Search Console 改善ワークフロー」を Monitor の定期タスクに登録してループを回す
⚠️ 注意: AIの出力はそのまま公開せず、必ず人間がレビューしてから確定してください。数値・順位の断定や誇張表現は避け、事実は一次データで裏取りを。
