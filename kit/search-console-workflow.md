# 🎁 Search Console 改善ワークフロー

Search Console のデータを起点に、順位下落・低CTR・indexエラーを直す監視ループの手順書。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: Monitor エージェントの定期タスクとして登録する。データはAPI / 連携ツール経由で読ませる。

```text
# Search Console 改善ワークフロー

## 前提
Search Console のデータ（順位・CTR・表示回数・index）を
エージェントが読める状態にする（API / 連携ツール経由）。

## 定期チェック（例: 週1）
1. 掲載順位が下落したクエリ / ページを抽出（前週比）
2. 表示回数は多いがCTRが低いページを抽出（改善余地大）
3. index エラー・カバレッジ問題を抽出

## 改善アクション
- 低CTRページ -> タイトル / メタディスクリプションを検索意図に合わせて書き換え
- 順位下落ページ -> 競合の更新点を調べ、該当箇所を改稿（既存記事改善ループへ）
- index エラー -> 原因を分類し、対応 or エスカレーション

## 記録
- 変更前後の指標を LOG に残し、2〜4週間後に効果を再測定する

## 注意
- 一度に多くを変えず、変更点を絞って因果を測れるようにする
- 連携ツールはサードパーティの場合がある。権限は最小限に
```

**ポイント**: 変更は一度に絞る。まとめて変えると、何が効いたか分からなくなる。
